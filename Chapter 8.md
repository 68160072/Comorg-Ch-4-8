## แบบฝึกหัด (Problems)

**8.1** สมมติว่าเรามีคอมพิวเตอร์แบบมัลติโปรแกรมที่แต่ละงานมีลักษณะเหมือนกัน ในหนึ่งรอบการคำนวณ T สำหรับงานหนึ่ง ครึ่งหนึ่งของเวลาถูกใช้ใน I/O และครึ่งหนึ่งในกิจกรรมโปรเซสเซอร์ แต่ละงานทำงานรวมทั้งหมด N รอบ สมมติว่าใช้ลำดับความสำคัญแบบ round-robin อย่างง่าย และการดำเนินการ I/O สามารถทับซ้อนกับการดำเนินการโปรเซสเซอร์ได้ กำหนดปริมาณต่อไปนี้:

- **Turnaround time** = เวลาจริงในการทำงานให้เสร็จ
- **Throughput** = จำนวนงานเฉลี่ยที่เสร็จสิ้นต่อรอบเวลา T
- **Processor utilization** = เปอร์เซ็นต์ของเวลาที่โปรเซสเซอร์ทำงานอยู่ (ไม่รอ)

คำนวณปริมาณเหล่านี้สำหรับงานหนึ่ง สอง และสี่งานพร้อมกัน สมมติว่ารอบ T ถูกแจกแจงในแต่ละวิธีดังต่อไปนี้:

- **a.** I/O ครึ่งแรก, โปรเซสเซอร์ครึ่งหลัง
- **b.** I/O ไตรมาสแรกและสี่, โปรเซสเซอร์ไตรมาสสองและสาม

---

**8.2** โปรแกรมที่ผูกกับ I/O (I/O-bound) คือโปรแกรมที่ถ้าทำงานคนเดียวจะใช้เวลารอ I/O มากกว่าใช้โปรเซสเซอร์ โปรแกรมที่ผูกกับโปรเซสเซอร์ (processor-bound) คือสิ่งตรงข้าม สมมติว่าอัลกอริทึมการจัดตารางเวลาระยะสั้นชอบโปรแกรมที่ใช้เวลาโปรเซสเซอร์น้อยในช่วงที่ผ่านมาเมื่อเร็วๆ นี้ อธิบายว่าทำไมอัลกอริทึมนี้จึงชอบโปรแกรมที่ผูกกับ I/O แต่ยังไม่ปฏิเสธเวลาโปรเซสเซอร์แก่โปรแกรมที่ผูกกับโปรเซสเซอร์อย่างถาวร

---

**8.3** โปรแกรมคำนวณผลรวมแถว:

$$C_i = \sum_{j=1}^{n} a_{ij}$$

ของอาร์เรย์ A ขนาด 100 × 100 สมมติว่าคอมพิวเตอร์ใช้ demand paging ที่มีขนาดหน้า 1,000 คำ และปริมาณหน่วยความจำหลักที่จัดสรรให้ข้อมูลคือ 5 page frames มีความแตกต่างในอัตราข้อผิดพลาดหน้าหรือไม่ถ้า A ถูกจัดเก็บในหน่วยความจำเสมือนตามแถวหรือคอลัมน์? อธิบาย

---

**8.4** พิจารณาแนวทางการแบ่งพาร์ทิชันขนาดคงที่ที่มีพาร์ทิชันขนาดเท่ากัน 2^16 ไบต์ และขนาดหน่วยความจำหลักทั้งหมด 2^24 ไบต์ มีการรักษาตารางกระบวนการซึ่งรวมถึงตัวชี้ไปยังพาร์ทิชันสำหรับแต่ละกระบวนการที่ประจำอยู่ จำเป็นต้องใช้กี่บิตสำหรับตัวชี้?

---

**8.5** พิจารณาแนวทางการแบ่งพาร์ทิชันแบบไดนามิก แสดงว่าโดยเฉลี่ย หน่วยความจำบรรจุช่องว่างครึ่งหนึ่งเมื่อเทียบกับเซกเมนต์

**8.6** สมมติว่าตารางหน้าสำหรับกระบวนการที่กำลังทำงานบนโปรเซสเซอร์มีลักษณะดังต่อไปนี้ ตัวเลขทั้งหมดเป็นเลขฐานสิบ ทุกอย่างถูกนับเริ่มจากศูนย์ และที่อยู่ทั้งหมดเป็นที่อยู่ไบต์หน่วยความจำ ขนาดหน้าคือ 1,024 ไบต์

| Virtual page number | Valid bit | Reference bit | Modify bit | Page frame number |
|---------------------|-----------|---------------|------------|-------------------|
| 0 | 1 | 1 | 0 | 4 |
| 1 | 1 | 1 | 1 | 7 |
| 2 | 0 | 0 | 0 | — |
| 3 | 1 | 0 | 0 | 2 |
| 4 | 0 | 0 | 0 | — |
| 5 | 1 | 0 | 1 | 0 |

**a.** อธิบายอย่างแม่นยำว่าโดยทั่วไปแล้ว ที่อยู่เสมือนที่สร้างโดย CPU ถูกแปลเป็นที่อยู่หน่วยความจำหลักจริงได้อย่างไร

**b.** ที่อยู่จริงใด (ถ้ามี) ที่ที่อยู่เสมือนต่อไปนี้แต่ละอย่างสอดคล้องกับ? (อย่าพยายามจัดการกับ page fault ใดๆ ถ้ามี)

- ii. 2221
- iii. 5499

**8.7** ให้เหตุผลว่าทำไมขนาดหน้าในระบบหน่วยความจำเสมือนไม่ควรเล็กเกินไปหรือใหญ่เกินไป

**8.8** กระบวนการอ้างอิงหน้าห้าหน้า A, B, C, D และ E ในลำดับต่อไปนี้:

```
A; B; C; D; A; B; E; A; B; C; D; E
```

สมมติว่าอัลกอริทึมการแทนที่คือ first-in-first-out และหาจำนวนการถ่ายโอนหน้าระหว่างลำดับการอ้างอิงนี้ เริ่มต้นด้วยหน่วยความจำหลักว่างเปล่าที่มีสาม page frames ทำซ้ำสำหรับสี่ page frames

**8.9** ลำดับต่อไปนี้ของหมายเลขหน้าเสมือนถูกพบระหว่างการทำงานบนคอมพิวเตอร์ที่มีหน่วยความจำเสมือน:

```
3 4 2 6 4 7 1 3 2 6 3 5 1 2 3
```

สมมติว่านโยบายการแทนที่หน้า least recently used (LRU) ถูกนำมาใช้ วาดกราฟของอัตราการโดนแคช (page hit ratio = เศษส่วนของการอ้างอิงหน้าที่หน้าอยู่ในหน่วยความจำหลัก) เทียบกับความจุหน้าหน่วยความจำหลัก n สำหรับ 1 ≤ n ≤ 8 สมมติว่าหน่วยความจำหลักเริ่มต้นว่างเปล่า

**8.10** ในคอมพิวเตอร์ VAX ตารางหน้าผู้ใช้อยู่ที่ที่อยู่เสมือนในพื้นที่ระบบ ข้อได้เปรียบของการมีตารางหน้าผู้ใช้ในหน่วยความจำเสมือนแทนหน่วยความจำหลักคืออะไร? ข้อเสียคืออะไร?

**8.11** สมมติว่าคำสั่งโปรแกรม:

```c
for(i = 1; i <= n; i++)
    a[i] = b[i] + c[i];
```

ถูกทำงานในหน่วยความจำที่มีขนาดหน้า 1,000 คำ ให้ n = 1,000 โดยใช้เครื่องที่มีคำสั่ง register-to-register แบบครบชุดและใช้ index registers เขียนโปรแกรมสมมติเพื่อใช้งานคำสั่งข้างต้น จากนั้นแสดงลำดับของการอ้างอิงหน้าระหว่างการทำงาน

**8.12** สถาปัตยกรรม IBM System/370 ใช้โครงสร้างหน่วยความจำสองระดับและอ้างถึงสองระดับนั้นว่าเซกเมนต์และหน้า แม้ว่าแนวทางการแบ่งเซกเมนต์ขาดคุณลักษณะมากมายที่อธิบายไว้ในบทนี้ สำหรับสถาปัตยกรรม 370 พื้นฐาน ขนาดหน้าอาจเป็น 2 กิโลไบต์หรือ 4 กิโลไบต์ และขนาดเซกเมนต์คงที่ที่ 64 กิโลไบต์หรือ 1 เมกะไบต์ สำหรับสถาปัตยกรรม 370/XA และ 370/ESA ขนาดหน้าคือ 4 กิโลไบต์และขนาดเซกเมนต์คือ 1 เมกะไบต์ แนวทางนี้ขาดข้อได้เปรียบใดของการแบ่งเซกเมนต์? ประโยชน์ของการแบ่งเซกเมนต์สำหรับ 370 คืออะไร?

**8.16** สมมติว่าไมโครโปรเซสเซอร์ที่สามารถเข้าถึงหน่วยความจำจริงหลักได้ถึง 2^32 ไบต์ มันใช้งานพื้นที่ที่อยู่ตรรกะแบบมีเซกเมนต์หนึ่งพื้นที่ที่มีขนาดสูงสุด 2^31 ไบต์ แต่ละคำสั่งบรรจุที่อยู่สองส่วนทั้งหมด หน่วยจัดการหน่วยความจำภายนอก (External MMU) ถูกใช้ โดยแนวทางการจัดการมอบหมายบล็อกหน่วยความจำจริงที่ต่อเนื่องกันขนาดคงที่ 2^22 ไบต์ให้กับเซกเมนต์ ที่อยู่จริงเริ่มต้นของเซกเมนต์หารด้วย 1,024 เสมอ แสดงการเชื่อมต่อโดยละเอียดของกลไกการแมปภายนอกที่แปลงที่อยู่ตรรกะเป็นที่อยู่จริง โดยใช้จำนวน MMU ที่เหมาะสม และแสดงโครงสร้างภายในโดยละเอียดของ MMU (สมมติว่า MMU แต่ละตัวบรรจุ segment descriptor cache แบบ directly mapped 128 รายการ) และ MMU แต่ละตัวถูกเลือกอย่างไร

**a.** รูปแบบของที่อยู่ตรรกะของโปรเซสเซอร์คืออะไร?
**b.** ความยาวและความกว้างของตารางหน้าคือเท่าใด (โดยไม่นับบิต "สิทธิ์การเข้าถึง")?
**c.** ผลกระทบต่อตารางหน้าถ้าพื้นที่หน่วยความจำจริงลดลงครึ่งหนึ่งคืออะไร?

**8.18** ในระบบปฏิบัติการเมนเฟรม OS/390 ของ IBM หนึ่งในโมดูลหลักในเคอร์เนลคือ System Resource Manager (SRM) โมดูลนี้รับผิดชอบในการจัดสรรทรัพยากรระหว่าง address spaces (กระบวนการ) SRM ให้ OS/390 มีระดับความซับซ้อนที่ไม่เหมือนใครในบรรดาระบบปฏิบัติการ ไม่มีระบบปฏิบัติการเมนเฟรมอื่นใด และแน่นอนว่าไม่มี OS ประเภทอื่นใดที่จะสามารถเทียบเทียมกับฟังก์ชันที่ SRM ดำเนินการได้

แนวคิดของทรัพยากรรวมถึงโปรเซสเซอร์ หน่วยความจำจริง และช่องทาง I/O SRM สะสมสถิติเกี่ยวกับการใช้งานโปรเซสเซอร์ ช่องทาง และโครงสร้างข้อมูลสำคัญต่างๆ จุดประสงค์ของมันคือให้ประสิทธิภาพที่เหมาะสมที่สุดตามการตรวจสอบและการวิเคราะห์ประสิทธิภาพ การติดตั้งตั้งวัตถุประสงค์ประสิทธิภาพต่างๆ และสิ่งเหล่านี้ทำหน้าที่เป็นแนวทางให้ SRM ซึ่งปรับลักษณะการติดตั้งและงานแบบไดนามิกตามการใช้งานระบบ

ปัญหานี้เกี่ยวกับตัวอย่างหนึ่งของกิจกรรม SRM หน่วยความจำจริงถูกแบ่งออกเป็นบล็อกขนาดเท่ากันที่เรียกว่า **เฟรม (frames)** ซึ่งอาจมีหลายพันเฟรม แต่ละเฟรมสามารถบรรจุบล็อกของหน่วยความจำเสมือนที่เรียกว่า **หน้า (page)** SRM ได้รับการควบคุมประมาณ 20 ครั้งต่อวินาทีและตรวจสอบแต่ละ page frame ทุกครั้ง ถ้าหน้าไม่ได้ถูกอ้างอิงหรือเปลี่ยนแปลง ตัวนับจะเพิ่มขึ้น 1 ตามกาลเวลา SRM เฉลี่ยตัวเลขเหล่านี้เพื่อกำหนดจำนวนวินาทีเฉลี่ยที่ page frame ในระบบไม่ถูกแตะต้อง จุดประสงค์อาจเป็นอะไร และ SRM อาจดำเนินการอะไร?
